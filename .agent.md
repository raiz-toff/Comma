# Comma App Design Rules

The following design rules MUST be strictly followed for all new screens, UI components, and layout modifications. These rules ensure that the application maintains a cohesive, premium, and unified aesthetic across all tabs (Shifts, Analytics, Expenses, etc.).

## 1. True OLED Dark Theme
- **Root Backgrounds**: Always use pure OLED pitch black (`#000000`) for screen backgrounds, `SafeAreaView`s, and base containers.
- **Surface Cards**: Use a very deep slate gray (`#0d0d0d`) for individual cards, lists, Bento boxes, and elevated surfaces. Do NOT use lighter silvers like `#161615` for main content cards.
- **Surface Borders**: Use `#1f1f1f` for borders on `#0d0d0d` surfaces. The `borderWidth` should typically be `0.8`.
- **Button/Pill Surfaces**: Secondary elements like filter pills, date selectors, and arrow buttons can use `#161615` to stand out slightly from the OLED black background, accompanied by a `#262522` border (thickness `0.8`).

## 2. Perfect Pill Geometry (Curves)
- **Border Radii**: There are no "semi-rounded" rectangles in the top navigation elements. Everything is either a perfect pill or a circle.
  - Date / Month / Week selector pills: `borderRadius: 20` (minimum height 32-40).
  - Navigation arrow buttons (e.g. `ChevronLeft`, `ChevronRight`): `width: 44, height: 44, borderRadius: 22`.
  - Filter category chips/pills: `borderRadius: 20`.
- Do NOT use `borderRadius: 10` or `12` for top-header filter pills. Always opt for the fully rounded `20` curve.

## 3. Huge, Ultra-Bold Typography
- Numerical data must be massive, easily readable at a glance, and ultra-bold.
- **Main Hero Numbers (e.g. Total Income, Net Income)**:
  - Value text: `fontSize: 40`, `fontWeight: "800"`, `color: "#ffffff"`, `letterSpacing: -0.5`, `lineHeight: 48`. Always use `adjustsFontSizeToFit` and `numberOfLines={1}`.
  - Currency symbol: Must be visually distinct. Use `fontSize: 24`, `fontWeight: "600"`, and place it beside the value text, top-aligned using `flexDirection: "row"`. Always separate the symbol using `Intl.NumberFormat` parts.
- **Bento Grid / Stat Cards**:
  - Number values: `fontSize: 32`, `fontWeight: "800"`, `color: "#ffffff"`, `letterSpacing: -1`. Include `adjustsFontSizeToFit`.
- **Labels**: 
  - Standard small headers: `fontSize: 11` or `12`, `fontWeight: "800"`, text-transform `uppercase`, wide letter spacing (e.g. `0.5` or `1`).

## 4. Seamless Tab Navigation (Layout Structure)
- **Preventing Layout Flicker**: When structuring scrollable screens, do NOT apply safe-area `paddingTop` to the outer `<SafeAreaView>`. 
- **Correct Layout Pattern**: Apply the top inset spacing (e.g., `paddingTop: insets.top + 64`) directly to the `<ScrollView contentContainerStyle={{...}}>`. 
- The `<View>` for the screen header (Date selector + navigation arrows) should live *inside* the `ScrollView` so it floats naturally, rather than being a static sticky block with a hard bottom border.
- **Syntax Integrity**: When refactoring screens to float headers inside `ScrollView` components, explicitly ensure that any newly introduced padding or margin `<View>` wrappers are properly closed before the terminating `</ScrollView>` tag to prevent syntax crashes.

## 5. Chart & Graph Aesthetics
- **Graph Container**: Like all Bento cards, the main container wrapping a chart must use `backgroundColor: "#0d0d0d"`, `borderRadius: 20`, `borderWidth: 0.8`, and `borderColor: "#1f1f1f"`.
- **Bar Tracks**: The background track for a vertical bar chart column must be slim and elegant. Use `width: 14` and `backgroundColor: "#161615"` (with a `borderRadius: 7`). Do NOT use thick `20px` bars or `#0d0d0d` tracks.
- **Overlays / Badges**: High-value dashed-line overlays and their text badges should blend seamlessly into the card background, utilizing `backgroundColor: "#0d0d0d"` to match the container surface.
